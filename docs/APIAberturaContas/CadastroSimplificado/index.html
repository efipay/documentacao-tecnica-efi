Cadastro Simplificado

O conjunto de endpoints abaixo são destinados a lidar com o Cadstro de uma conta simplificada

Solicitar abertura de conta simplificada.​

Solicita abertura de conta simplificada para integração. Os dados de entrada são dados referentes ao cliente final. Caso os dados de entrada sejam válidos, o cliente final receberá um link no qual poderá autorizar integração à uma conta Efí.

Criação de conta/aplicação
Caso o seu cliente final já tenha uma conta na Efí, duas opções vão aparecer para ele:
  • Criar uma aplicação na conta atual, onde você terá acesso às credenciais e definirá os escopos.
  • Criar uma conta secundária para o cliente final com as configurações/acessos definidos em sua requisição.

Negação da abertura de conta
Caso o seu cliente recuse o processo de abertura de conta, será necessário esperar 2 dias corridos até poder iniciar um novo processo.
Obs: Não é retornado a informação caso o cliente negue o processo de abertura.

POST /cadastro/conta-simplificada
Requer autorização para o escopo: gn.registration.write

Requisição

  • Cliente final pessoa jurídica

Recuperar credenciais de uma conta simplificada.​

Recupera as credenciais Client_Id e Client_Secret para integrar à uma conta simplificada.

GET /cadastro/conta-simplificada/:identificador/credenciais
Requer autorização para o escopo: gn.registration.read


Respostas

As respostas abaixo representam Sucesso(200) e Falhas/erros do consumo.

  • 200(Success)
  • 401(Unauthorized)
  • 403(Forbidden)
  • 412(Precondition Failed)
  • 500(Error)
{
"clientId": "92ccd29b-54c9-49fc-b8e8-717a3b373c5e",

  "webhook.read"

* Certificado ou credenciais não existem;
* Certificado ou credenciais estão desativados;
* Certificado e credenciais não estão vinculados a mesma conta Gerencianet;
* Integrador não tem permissão para o escopo de serviço necessário para consumir este endpoint.
Integrador tenta recuperar as credenciais de uma conta simplificada não criada por ele.
